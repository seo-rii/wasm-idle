# Upload Checklist

1. Confirm official source URLs and archive hashes or saved paths.
2. Confirm local nojam root and target contest root.
3. Validate `problems.json` structure.
4. Validate image files render and are referenced.
5. Validate test case suffixes and input/output pairing.
6. Add or verify accepted solutions.
7. Add official SPJ/checkers or mark ordinary judge.
8. Add interactive interactor, placeholder, language limits, and local transcript tests.
9. Run structural validator.
10. Run local compile/test commands.
11. Search remote by exact `from` and verify duplicate mappings.
12. Ask the user for starting Jungol ID.
13. Write `log/id.log`.
14. Dry-run upload.
15. Upload.
16. Submit accepted solutions.
17. Poll and report verdicts.
